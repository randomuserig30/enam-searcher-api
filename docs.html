Getting Started

Introduction

Welcome to the Enam Searcher API documentation. Our API allows you to programmatically search through billions of leaked records across thousands of data breaches.

Base URL

All API requests should be made to:

https://api.enamsearcher.com/v2

Overview

The Enam Searcher API is organized around REST. Our API accepts JSON-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes and authentication.

RESTful API with JSON responses
API key authentication via headers
Search by email, username, IP, phone, name, password, hash
Real-time database with 12B+ records
Rate limiting with clear headers
Getting Started

Authentication

The Enam Searcher API uses API keys to authenticate requests. You can view and manage your API keys in your Dashboard.

Keep your API key secure

Do not share your API key in publicly accessible areas such as GitHub, client-side code, or public repositories.

Using your API Key

Include your API key in the request header:

HTTP Header 
Authorization: Bearer YOUR_API_KEY

Example Request

cURL 
curl -X GET "https://api.enamsearcher.com/v2/search?type=email&query=test@example.com" \
  -H "Authorization: Bearer es_live_abc123def456"
Getting Started

Quick Start

Get up and running with the Enam Searcher API in minutes.

1

Create an Account

Sign up at enamsearcher.com and navigate to the API section in your dashboard.

2

Get your API Key

Generate a new API key from your dashboard. Copy it and store it securely.

3

Make your First Request

Use the example below to perform your first search:

Python 
import requests

API_KEY = "es_live_abc123def456"
BASE_URL = "https://api.enamsearcher.com/v2"

headers = {
    "Authorization": f"Bearer {API_KEY}"
}

response = requests.get(
    f"{BASE_URL}/search",
    headers=headers,
    params={
        "type": "email",
        "query": "test@example.com"
    }
)

print(response.json())
API Reference

Search

Search across all databases for matching records.

GET /v2/search

Query Parameters

Parameter Type Required Description
type string Required Search type: email, username, ip, phone, name, password, hash
query string Required The search term to look for
limit integer Optional Max results to return (default: 100, max: 1000)
offset integer Optional Pagination offset (default: 0)

Response

JSON Response — 200 OK 
{
  "success": true,
  "took": 0.042,
  "total": 3,
  "results": [
    {
      "source": "LinkedIn.com",
      "date": "2023-06",
      "fields": {
        "email": "test@example.com",
        "password": "hashed_password",
        "name": "John Doe",
        "ip": "192.168.1.1"
      }
    }
  ]
}

Resources

Error Codes

The API uses standard HTTP status codes to indicate the success or failure of requests.

Code Status Description
200 OK Request was successful
400 Bad Request Invalid parameters or missing required fields
401 Unauthorized Missing or invalid API key
403 Forbidden API key does not have access to this resource
429 Too Many Requests Rate limit exceeded
500 Server Error Internal server error, try again later

Error Response Format

JSON Error — 401 Unauthorized 
{
  "success": false,
  "error": {
    "code": 401,
    "message": "Invalid API key provided."
  }
}

Code Examples

Ready-to-use code snippets in popular programming languages.

Python — requests 
import requests

API_KEY = "es_live_abc123def456"

response = requests.get(
    "https://api.enamsearcher.com/v2/search",
    headers={"Authorization": f"Bearer {API_KEY}"},
    params={
        "type": "email",
        "query": "target@example.com",
        "limit": 50
    }
)

data = response.json()
for result in data["results"]:
    print(f"Source: {result['source']}")
    print(f"Fields: {result['fields']}")
    print("---")
JavaScript — fetch

const response = await fetch(
  "https://api.enamsearcher.com/v2/search?type=email&query=target@example.com",
  {
    headers: {
      "Authorization": `Bearer ${API_KEY}`
    }
  }
);

const data = await response.json();
console.log(data.results);
cURL 
curl -X GET \
  "https://api.enamsearcher.com/v2/search?type=email&query=target@example.com&limit=50" \
  -H "Authorization: Bearer es_live_abc123def456" \
  -H "Content-Type: application/json"
PHP — cURL 
$apiKey = "es_live_abc123def456";
$url = "https://api.enamsearcher.com/v2/search?type=email&query=target@example.com";

$ch = curl_init($url);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    "Authorization: Bearer " . $apiKey
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
$data = json_decode($response, true);

print_r($data["results"]);
